Background

What Is an
Adversarial Patch?

The idea

A small image region, optimized through gradient descent, that causes a neural network to stop detecting objects — even when the object is fully visible.

How it's different from occlusion

Covering a face with any object reduces features. An adversarial patch exploits the model's internal representations — creating activation patterns the detector interprets as "no person."

Training loop

  • Freeze model weights
  • Overlay patch on person's torso
  • Forward pass → detection confidence
  • Backpropagate to patch pixels only
  • Repeat until detections suppress

Robustness augmentations

  • Random rotation ±15°
  • Cutout masking (T-SEA)
  • Block erasing (DePatch)
  • Non-Printability Score loss

Why

Adversarial patches concentrate signal in a dense region. Unlike distributed pixel perturbations, the patch survives lossy operations — blur suppresses high-frequency clean texture while leaving the patch's dominant structure intact. Standard preprocessing defenses are not designed for this threat model.

Key Research Finding

The v26n Paradox

What happened

Optimizer fully converged.
final_det_loss = 0.108 — lower than v8n (0.543) and v11n (0.597).
Suppression: only 16.3%.

Root cause

YOLO26n uses end-to-end Hungarian matching (head_end2end: true).

  • Training optimizes the one2many auxiliary head
  • Inference uses the one2one head for final detections
  • These heads are architecturally separate

Three experiments — same answer

ExperimentLossSuppression
Cold start · one2many loss0.10816.3%
Warm start from v8n 90% patch0.10314.0%
Cold start · one2one loss (v3)0.09411.6%

Better loss convergence → less suppression. The relationship is inverted.

Conclusion

YOLO26n is a natural defense against this attack class. The Hungarian matching layer fully decouples loss from inference output — targeting either head in isolation cannot overcome it. No existing patch attack is designed for this architecture.

Conclusions

What We
Learned

Patches work

A 2.4% image patch suppresses YOLOv8n person detection by 90%. The effect transfers across model families at 10–50%.

Architecture matters

YOLO26n's end-to-end Hungarian matching head creates a structural barrier. Loss convergence does not equal detection suppression.

Open questions

Does the NMS-free shift create a new attack class, or does it port from DETR literature? What attack formulation breaks through the Hungarian matching layer? Physical robustness at scale.
